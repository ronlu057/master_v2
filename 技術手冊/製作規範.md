## 1. 一般原則

### ✅ DO

- 開工前先讀 [開發母版專案.md](./開發母版專案.md) 與本檔
- 命名統一英文（PascalCase 元件、camelCase 變數、kebab-case CSS class）
- 改任何「全站行為」（如 `.env` 開關、`nuxt.config.js`、`app.vue`）前，**確認影響範圍**
- 對「為什麼這樣寫」不確定時，去白皮書找對應章節
- 增刪檔案要連動更新 mock 資料 / 設定 / 文件

### ❌ DON'T

- **禁用 TypeScript**（白皮書 [§9.1](./開發母版專案.md#91-vue-元件規範) — 嚴禁 `.ts` 檔或 `<script lang="ts">`）
- 不要在元件內 import Google Fonts CSS（一律走 `nuxt.config.js`）
- 不要自行變更白皮書版本號 / 修訂紀錄（除非客戶明確要求 bump）
- 不要硬填客戶資料進元件（用 mock JSON / firm 資料）
- 不要繞過派發器（AppHeader / PageBanner / BlockBanner）直接 import 子版型

## 2. Vue 元件規範

### 2.1 Script setup

✅ **必須**用 `<script setup>` 語法：

```vue
<script setup>
const { isShop } = useProject()
const cart = useCartStore()
</script>
```

❌ **禁止**用 Options API 或 `<script>` 沒 setup：

```vue
<!-- 禁 -->
<script>
export default {
  setup() { ... }
}
</script>
```

### 2.2 Props 宣告

✅ 用 `defineProps()` 明確宣告每個 prop：

```js
const props = defineProps({
  title: { type: String, default: '' },
  rows: { type: Array, default: () => [] },
})
```

❌ 不要用陣列簡寫（會少掉 type / default）：

```js
// 不建議
const props = defineProps(['title', 'rows'])
```

### 2.3 命名

| 對象 | 規則 | 範例 |
|---|---|---|
| 元件檔名 | **PascalCase 多字詞** | `NewsCard.vue`、`BlockNews.vue`、`AppHeader.vue` |
| 派發器子版型 | **前綴 + 數字** | `Header01.vue`、`PageBanner01.vue`、`BlockBanner02.vue` |
| Composable | **小駝峰 + use 前綴** | `useProject.js`、`useSiteMenu.js` |
| Pinia store | **小駝峰 + use 前綴 + Store 後綴** | `useUiStore.js`、`useCartStore.js` |
| Utils | **小駝峰** | `formatPrice.js`、`parseBool.js` |

> [!CAUTION]
> 單字元件名（如 `Card.vue`、`Logo.vue`）會跟 HTML 標籤衝突，Vue 會警告。**一律用多字詞**。

### 2.4 資料獲取

| 場景 | 用什麼 |
|---|---|
| 頁面 / 元件 setup 階段取 SSR 資料 | `useApiData('/path')`（白皮書 [§7.1](./開發母版專案.md#71-統一串接層)） |
| 表單送出、按鈕觸發等命令式請求 | `$api('/path', { method: 'POST', body })` |
| 全站共用資料（選單 / 公司 / 多語） | `useSiteMenu()` / `useSiteFirm()` / `useDict()` |

❌ **不要**直接用 `$fetch` 或 `fetch()` 繞過 `useApi.js`：

```js
// 禁 — 不會走統一格式 unwrap
const data = await $fetch('/api/news/list')
```

### 2.5 元件生命週期

✅ 監聽器、interval、ScrollTrigger 等**必須**在 `onBeforeUnmount` 清理：

```js
const id = setInterval(...)
window.addEventListener('scroll', handler)

onBeforeUnmount(() => {
  clearInterval(id)
  window.removeEventListener('scroll', handler)
})
```

詳見白皮書 [§9.11.6](./開發母版專案.md#9116-gsap--scrolltrigger-用法)（含 ScrollTrigger 範例）。

## 3. 樣式（SCSS）規範

### 3.1 全站規範（白皮書 §9.4 IMPORTANT）

✅ **必須**：
- 所有 `<style>` 加 `lang="scss"`
- 用 `<style lang="scss" scoped>` 避免全域污染
- 主題色用 CSS 變數 `var(--color-primary)` 等

❌ **嚴禁**：
- 純 CSS（無 lang="scss"）
- 元件內 `style="..."` 行內樣式
- 獨立 `.css` 檔
- 用 `!important`（除非有極特殊覆寫第三方需求）
- **BEM `&__` 巢狀寫法**（詳見下方 §3.1.1）

#### 3.1.1 禁用 BEM `&__` 巢狀寫法

❌ **禁止**用 SCSS 的 `&` 父選擇器組 BEM child class：

```scss
// 禁
.card {
  padding: 20px;

  &__title {       // ❌ 編譯後變 .card__title
    font-size: 20px;
  }
  &__body {        // ❌ 同上
    color: #333;
  }
}
```

✅ **必須**寫完整 class 名稱（攤平結構）：

```scss
// 推薦
.card {
  padding: 20px;
}
.card__title {     // ✓ 直接寫完整 class
  font-size: 20px;
}
.card__body {      // ✓
  color: #333;
}
```

**為什麼**：
- **可搜尋性高**：`grep .card__title` 直接找到定義，不用先找 `.card { &__title }`
- **重構不易出錯**：改 `.card` 為 `.product-card` 時，`&__` 衍生 class 會跟著變，可能造成「外面用 .card__title 但 SCSS 已變 .product-card__title」斷裂
- **編譯結果相同**：兩種寫法輸出 CSS 完全一樣，沒效能差異
- **降低 SCSS 學習負擔**：新進工程師看完整 class 就懂，不用懂 `&` 上下文

> [!IMPORTANT]
> **判斷標準**：`&` 後面是「**衍生新 class 名稱**」就禁；「**附加既有條件**」就 OK。
>
> | 寫法 | 編譯後 | 判定 |
> |---|---|---|
> | `&:hover` | `.parent:hover` | ✅ OK（偽類） |
> | `&::before` | `.parent::before` | ✅ OK（偽元素） |
> | `&.is-active` | `.parent.is-active` | ✅ OK（附加既有狀態 class） |
> | `&.show` / `&.scroll` / `&.active` | `.parent.show` 等 | ✅ OK（附加既有 class） |
> | `&[data-page="index"]` | `.parent[data-page="index"]` | ✅ OK（屬性選擇器） |
> | `&__title` | `.parent__title` | ❌ **禁** — 衍生新 class，難 grep |
> | `&--primary` | `.parent--primary` | ❌ **禁** — 同上 |
> | `&-large` | `.parent-large` | ❌ **禁** — 同上 |
>
> ```scss
> .btn {
>   color: #333;
>   &:hover { color: blue; }        // ✓
>   &.is-active { color: red; }     // ✓
>   &.show { display: block; }      // ✓
>
>   &--primary { ... }              // ❌ 改寫成 .btn--primary { ... }
>   &__icon { ... }                 // ❌ 改寫成 .btn__icon { ... }
> }
> ```
>
> **為什麼這樣分**：`grep .btn--primary` 能直接找到定義 — 但 `grep &--primary` 找不到（因為實際字串是 `&--primary`，且不知道 `&` 是什麼）。「附加既有 class」不會產生這個問題，因為兩個 class（如 `.btn` 跟 `.is-active`）都能單獨被 grep 找到。

#### 3.1.2 後代選擇器巢狀（`.parent { .child {} }`）✅ 允許

**這跟 BEM `&__` 是不同概念**，編譯結果也不同：

| 寫法 | 編譯後 CSS | 套用範圍 |
|---|---|---|
| `.parent { &__child {} }` ❌ | `.parent__child {}` | **單一 class**（child 是獨立 class） |
| `.parent { .child {} }` ✅ | `.parent .child {}` | **後代關係**（任何 .parent 內的 .child） |

✅ **建議優先用後代選擇器巢狀**（更精簡、層次清楚）：

```scss
// 推薦
.index_banner {
  position: relative;
  width: 100vw;
  height: 100vh;

  .swiper-slide {
    position: relative;
    overflow: hidden;

    &::after {            // ✓ &::after 用偽類
      content: '';
      ...
    }
  }

  picture {
    display: block;

    img {                 // ✓ 多層後代巢狀 OK
      object-fit: cover;
    }
  }

  .swiper-pagination {
    text-align: left;

    @include rwd-768 {    // ✓ media query 內巢狀
      text-align: center;
    }
  }
}
```

✅ 也可以寫攤平形式（沒禁，看程式風格偏好）：

```scss
// 也 OK，但較囉嗦
.index_banner { position: relative; ... }
.index_banner .swiper-slide { ... }
.index_banner .swiper-slide::after { ... }
.index_banner picture { ... }
.index_banner picture img { ... }
```

> [!IMPORTANT]
> **避免重複 rule**：同一個選擇器寫兩次（一次 nested、一次攤平）會造成 dead code 與意外覆寫。Lint 或 review 時請檢查。

**何時用哪種？**

| 場景 | 推薦寫法 |
|---|---|
| 元件內 BEM child class（自己定義的 class） | **攤平**（`.parent__child`），方便 grep |
| 第三方套件渲染的 DOM 後代（`.swiper-slide`、`.swiper-pagination` 等） | **巢狀** `.parent { .child {} }`，限定作用範圍 |
| 偽類 / 狀態 | `&:hover`、`&.is-active`（`&` 偽類用法 OK） |
| 偽元素 | `&::before`、`&::after`（OK） |
| Media query | `@include rwd-XXX { ... }` 內巢狀（OK） |

### 3.2 SCSS 變數與 mixin

✅ 全域 SCSS 變數與 RWD mixin 來源：[`assets/styles/_variables.scss`](../assets/styles/_variables.scss)

由 [`nuxt.config.js`](../nuxt.config.js) 的 `additionalData` 自動注入，**任何 `<style lang="scss">` 都能直接用**，不需 `@use`：

```scss
.box {
  color: $web_font_color;
  z-index: $z_header;
  @include rwd-1200 { width: 100%; }
}
```

❌ **不要**在元件 SCSS 內 `@use "../../basic/config" as *` —— 路徑已不存在，且 `additionalData` 已自動處理。

### 3.3 顏色變數選用

| 變數類型 | 場景 |
|---|---|
| **CSS 變數**（`var(--color-primary)`） | 跟著「專案類型」變色的元素（白皮書 [§9.4](./開發母版專案.md#94-樣式與主題)） |
| **SCSS 變數**（`$web_header_1`） | 固定不隨主題變的色（例 Header02 內部使用） |

✅ 新元件**建議優先用 CSS 變數**，跟主題系統一致。

### 3.4 RWD 斷點

兩種寫法都可以，依「斷點是否落在標準值」選用：

**10 個標準斷點**：`2560 / 1920 / 1680 / 1440 / 1280 / 1200 / 1024 / 768 / 480 / 360`
主斷點：**1200**（header 變更樣式主斷點）；**2560** 對應 4K 大螢幕

#### 標準斷點 → 用 `@include rwd-xxx` mixin（建議）

從 [`_variables.scss`](../assets/styles/_variables.scss) 全域注入，元件內直接使用、不需 `@use`：

```scss
.box {
  width: 25%;
  @include rwd-1440 { width: 33%; }
  @include rwd-1200 { width: 50%; }
  @include rwd-768  { width: 100%; }
}
```

**好處**：斷點數值統一管理在 `_variables.scss`，未來要調整 `1200px → 1280px` 之類，動一處即可全站連動。

#### 非標準斷點 → 寫 `@media` 沒問題

當設計需要的斷點不在 9 個標準值內（例如 `540px`、`1440px 但要特殊處理`），直接寫 `@media`：

```scss
.box {
  @media (max-width: 540px) { padding: 8px; }  // ✓ 客製斷點
  @media (orientation: landscape) { ... }       // ✓ 非寬度條件
  @media print { ... }                          // ✓ 列印
}
```

#### 唯一禁止：用 `@media` 寫**標準斷點數字**

```scss
// ❌ 禁 — 標準斷點數字 hard-coded，與全站脫鉤
.box {
  @media (max-width: 1200px) { width: 50%; }
}

// ✓ 改用 @include
.box {
  @include rwd-1200 { width: 50%; }
}
```

> [!TIP]
> 看到 `@media (max-width: 1200px)`、`(max-width: 768px)` 這種「**剛好等於標準斷點**」的寫法，改成 `@include rwd-XXX`。其他客製尺寸不用動。

### 3.5 樣式 hook：`data-project` / `data-page`

✅ 利用自動掛在 `<html>` / `<body>` 的屬性切換樣式：

```scss
// 整站類型差異
[data-project="shop"] .logo { font-weight: 900; }

// 個別頁面 / 區塊差異
body[data-page="index"] .header02 { background: transparent; }
body[data-page="news"]  .page-banner__bg { filter: brightness(0.7); }
```

❌ **不要**自己手動往 `<html>` / `<body>` 加 class（一律在 `app.vue` 集中設定）。

詳見白皮書 [§9.12](./開發母版專案.md#912-全站樣式-hookdata-project--data-page)。

## 5. 設定值（.env / config）規範

### 5.1 `.env` 規則

✅ **DO**：
- 所有值前後**不要有空格**
- 字串就是字串（`true` / `false` 仍是字串，要靠 `parseBool` 解析）
- 用註解區塊分組（如 `# ── Header 版型 ───────`）

❌ **DON'T**：
- 不要 commit 含敏感資料的 `.env`（檢查 .gitignore）
- 不要把不確定的設定直接放 `.env`（先在 `nuxt.config.js` 設預設值）

### 5.2 Boolean 環境變數

✅ **必須**用 `parseBool()` 解析：

```js
import { parseBool } from '~/composables/useProject' // 範例位置

enableShop: !!matrix.shop || parseBool(pub.enableShop),
```

❌ **不要**用 `!!`：

```js
// 錯！" false"（含空格）會變 true
enableShop: !!pub.enableShop,
```

詳見白皮書 [§9.13](./開發母版專案.md#913-env-boolean-值正確寫法)。

### 5.3 `runtimeConfig` 與 `.env` 對應

✅ 命名規則：

| .env | runtimeConfig.public |
|---|---|
| `NUXT_PUBLIC_PROJECT_TYPE` | `projectType` |
| `NUXT_PUBLIC_API_BASE` | `apiBase` |
| `NUXT_PUBLIC_ENABLE_SHOP` | `enableShop` |
| `NUXT_PUBLIC_HEADER` | `header` |

> Nuxt 自動把 `NUXT_PUBLIC_XXX_YYY` 轉成 `xxxYyy` 小駝峰格式。

## 6. SVG / 圖片 / 字體規範

### 6.1 SVG 放哪

| 用法 | 位置 | 引入方式 |
|---|---|---|
| 元件內用、要 CSS 控 fill（純向量） | `assets/icon/` | `import Xxx from '...?component'` → `<Xxx />` |
| 含 base64 嵌入圖（假 SVG） | `assets/icon/` 或 `public/img/` | `?url` 或 `<img src="...">` |
| 需要固定 URL 對外（favicon、OG 圖） | `public/` | `<img src="/img/...">` |

### 6.2 SVG 清理

✅ Adobe Illustrator 匯出的 SVG 通常含 `<defs><style>.st0 { fill: #xxx }</style></defs>`，這會**鎖死** fill 顏色。匯入專案後請清掉這段：

```bash
# 批次清理腳本（不會動含品牌色的 SVG，自己列檔名）
node -e "..."
```

❌ **不要清**含品牌色的 SVG（LINE logo、Facebook 等），它們的顏色就是品牌規範。

詳見白皮書 [§9.4](./開發母版專案.md#94-樣式與主題)。

### 6.3 字體

✅ 從 [Google Fonts](https://fonts.google.com) 透過 [`nuxt.config.js`](../nuxt.config.js) 的 `app.head.link` 載入。

❌ **不要**在元件 / SCSS 內 `@import url(fonts.googleapis.com/...)`。

✅ 引用字體名稱時用 [`_variables.scss`](../assets/styles/_variables.scss) 內的變數：

```scss
font-family: $web_font;          // ✓
font-family: 'Noto Sans TC';     // ✗ 寫死字型名
```

詳見白皮書 [§9.9 Google Fonts](./開發母版專案.md#99-google-fonts-字型)。

## 7. 套件使用規範（Lenis / GSAP / Pinia）

### 7.1 Lenis（平滑捲動）

✅ 自動全站接管，**不需手動觸發**。

✅ 需要程式式捲動時用 `$lenis.scrollTo(目標)`：

```vue
<script setup>
const { $lenis } = useNuxtApp()
</script>
<button @click="$lenis.scrollTo(0)">↑ 回頂</button>
```

❌ **不要**用 `window.scrollTo()` / `element.scrollIntoView()`（會跟 Lenis 衝突）。

❌ **不要**用 `document.body.style.overflow = 'hidden'` 鎖背景捲動 —— 用 `$lenis.stop()`。

詳見白皮書 [§9.11](./開發母版專案.md#911-平滑捲動lenis--gsap)。

### 7.2 GSAP / ScrollTrigger

✅ 元件內取出：

```js
const { $gsap, $ScrollTrigger } = useNuxtApp()
```

✅ ScrollTrigger 必須在 `onMounted` 內呼叫，並在 `onBeforeUnmount` `kill()`：

```js
onMounted(() => {
  $gsap.from(target.value, { scrollTrigger: { trigger: target.value, ... } })
})
onBeforeUnmount(() => {
  $ScrollTrigger.getAll().forEach((st) => {
    if (st.vars.trigger === target.value) st.kill()
  })
})
```

詳見白皮書 [§9.11.6](./開發母版專案.md#9116-gsap--scrolltrigger-用法)。

### 7.3 Pinia store

✅ 採 **Setup Store** 風格：

```js
export const useCartStore = defineStore('cart', () => {
  const items = ref([])
  const count = computed(() => items.value.length)
  function add(item) { items.value.push(item) }
  return { items, count, add }
})
```

❌ **不要**用 Options Store 風格（state / getters / actions 物件式）。

詳見白皮書 [§9.2](./開發母版專案.md#92-狀態管理pinia)。

### 7.4 i18n（@nuxtjs/i18n）

✅ 取翻譯字串、語系狀態統一用 `useI18n()`：

```js
const { t, locale, locales, setLocale } = useI18n()
```

✅ 介面固定文字（按鈕、選單、提示）用 `t('key')`，從 [i18n/locales/](../i18n/locales/) 字典取。

❌ **不要**在元件內硬編碼 `languages` 陣列：

```js
// 禁 — 與 nuxt.config.js 重複，改一處要改兩處
const languages = [
  { code: 'tw', label: '繁體中文' },
  { code: 'en', label: 'English' },
]
```

✅ 一律用：

```js
const { locales } = useI18n()  // 從 nuxt.config.js i18n.locales 取
```

❌ **不要**手動寫死字串供翻譯：

```vue
<!-- 禁 — 切換語系時不會變 -->
<button>看更多</button>
```

✅ 一律用 `$t()`：

```vue
<button>{{ $t('btn.more') }}</button>
```

詳見白皮書 [§9.10](./開發母版專案.md#910-多語系nuxtjsi18n)。

### 7.5 Swiper（輪播）

✅ 用 [Swiper 11](https://swiperjs.com/) 官方 Vue 元件，**不要**用 jQuery 版的 swiper / slick / owl-carousel：

```vue
<script setup>
import { Swiper, SwiperSlide } from 'swiper/vue'
import { Autoplay, EffectFade, Pagination, Navigation, Controller } from 'swiper/modules'
import 'swiper/css'
import 'swiper/css/effect-fade'
import 'swiper/css/pagination'
import 'swiper/css/navigation'
</script>

<template>
  <Swiper
    :modules="[Autoplay, EffectFade, Pagination]"
    :slides-per-view="1"
    :loop="true"
    effect="fade"
    :autoplay="{ delay: 5000 }"
    :pagination="{ clickable: true }"
  >
    <SwiperSlide v-for="(row, i) in rows" :key="i">...</SwiperSlide>
  </Swiper>
</template>
```

✅ **modules 按需 import**（不要全部 import，會增加 bundle size）：每用一個功能就 import 對應 module + 對應 CSS。

✅ **雙 Swiper 同步**（例：主圖 + 文字 cover 一起切換）—— 用 `Controller` module + `controller.control`：

```js
import { Controller } from 'swiper/modules'

const mainSwiper = ref(null)
const coverSwiper = ref(null)
const onMainReady = (s) => {
  mainSwiper.value = s
  if (coverSwiper.value) {
    s.controller.control = coverSwiper.value
    coverSwiper.value.controller.control = s
  }
}
// cover 的 onSwiper 同樣處理
```

❌ **不要**用全域 jQuery `$('.banner').on(...)` 處理輪播事件，改用 Swiper Vue 元件的 `@slide-change` 等事件。

範例：[components/banners/BlockBanner01.vue](../components/banners/BlockBanner01.vue)。

### 7.6 jQuery 套件遷移對應表

由舊專案搬過來的 jQuery 套件**不要直接照用**，本專案改用 Vue 友好的替代：

| 舊 jQuery 套件 | Vue 友好替代 | 備註 |
|---|---|---|
| `jquery.swiper` / Slick / Owl Carousel | [swiper 11 / Vue 元件](https://swiperjs.com/vue) | 詳見 §7.5 |
| `jquery.mb.YTPlayer` | **YouTube iframe embed** | 用 `<iframe src="https://www.youtube.com/embed/<id>?autoplay=1&mute=1&loop=1&playlist=<id>&controls=0">`；要程式控制再考慮 [vue3-youtube](https://github.com/anteriovieira/vue3-youtube) |
| `jquery.lightgallery` | 純 CSS + Vue [Transition](https://vuejs.org/guide/built-ins/transition.html) 寫自己的 modal，或裝 [@vueform/multiselect](https://github.com/vueform/) 等 Vue 套件 | LightGallery 也有 Vue 版本，但有授權限制 |
| `magnific-popup` | 同上，自寫 modal | |
| `bxslider` | swiper 11 | |
| `wow.js` / `aos.js` | GSAP ScrollTrigger（詳見 §7.2） | |
| jQuery scroll 偵測 | `window.addEventListener('scroll', ...)` + Vue ref | 範例：[Header01.vue 的 isScrolled](../components/headers/Header01.vue) |
| jQuery `$(...).addClass / removeClass` | `:class="{ active: x }"` 綁定 reactive state | |
| jQuery `$(window).on('load')` | `onMounted(...)` | |
| YTPlayer 控制 | iframe + `postMessage`，或 vue3-youtube | |

> [!IMPORTANT]
> **設計師交付 jQuery 樣板給工程師時**：把 JS 邏輯拆出來看，照本表轉成 Vue 等價寫法。**不要**為了快直接 `import 'jquery'` —— 一旦引入 jQuery，Vue 的 reactive 系統會跟 jQuery DOM 操作衝突（jQuery 改 DOM 後 Vue 不知道）。

## 8. Git / Commit / PR 規範

### 8.1 Commit message

✅ 範例（觀察 git log 已有的風格）：

```
feat: 新增 Banner 版型派發機制
fix: 修正 Header02 下拉動畫不對齊
refactor: 把語系切換抽到 useLangStore
docs: 更新白皮書 §12 Banner 版型切換
chore: 升級 Lenis 至 1.3.23
```

✅ 動詞分類：
- `feat:` 新功能
- `fix:` 修 bug
- `refactor:` 重構（不改行為）
- `docs:` 只動文件
- `chore:` 雜項（套件升級、設定調整）
- `style:` 純樣式修改

❌ 不要寫含糊 message：

```
update     // 太空泛
fix bug    // 哪個 bug？
asdf       // 無意義
```

### 8.2 文件同步

✅ 改 `.md` 後務必執行：

```bash
npm run docs
```

把 HTML 同步產生，commit 時兩個檔（.md + .html）一起。

❌ 不要只 commit .md 不 commit .html（線上閱讀靠 HTML）。

### 8.3 不要 push 到 main 前的 checklist

| 檢查 | 指令 |
|---|---|
| 沒有 console.log / debugger | `grep -rn "console.log" components/ pages/` |
| 沒有 TODO 未處理 | `grep -rn "TODO" components/ pages/` |
| build 沒錯 | `npm run build` |
| docs 已同步 | `npm run docs` |
